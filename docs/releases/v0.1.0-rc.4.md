# v0.1.0-rc.4

## Summary

This release candidate expands the Rust CLI from lookup and search into a fuller local retrieval surface. Search now uses richer SQLite-backed FTS ranking, structured filter pushdown, ranked-result hydration after fusion, and coverage-driven rich-content embeddings for better semantic recall without loading unnecessary result detail.

New and improved user-facing surfaces:

- `atlas graph links`, `atlas graph uses`, `atlas graph variants`, and `atlas graph remaster` provide local reference, backlink, variant, and remaster context around resolved records.
- `atlas similar <record>` finds records similar to a seed record using stored record embeddings plus shared reference and trait evidence.
- Record identity resolution now runs through SQLite, dedupes alias matches, and returns structured ambiguity alternatives for `record resolve`, graph seed resolution, and similar seed resolution in JSON.
- `atlas setup` and build/index commands have clearer progress output, including timing summaries and machine-readable not-ready reasons for setup diagnostics.
- The bundled `pf2e-atlas-cli` agent skill has updated command guidance for graph context, similar-record lookup, action records, and JSON ambiguity handling.
- Source loading and search result hydration are faster, and the workspace now has `just verify` and related contributor workflows.

## Install Notes

- Install or update the CLI normally, then run `atlas setup --check --json` to inspect whether the existing source checkout, embedding model cache, and SQLite artifact are ready.
- Run `atlas setup` after upgrading if setup reports a planned rebuild, stale source signature, missing embeddings, or incompatible artifact metadata. This rc includes artifact, embedding, and retrieval changes that can require a rebuilt local index.
- Existing scripts that parse JSON ambiguity errors should prefer `error.data.result.alternatives` when present instead of scraping candidate keys from `error.message`.
